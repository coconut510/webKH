@ 인터넷이란?
-> 전세계의 컴퓨터가 서로 연결되어 정보를 교환할 수 있는 하나의 거대한
컴퓨터 통신망
-> 인터넷에서는 다양한 서비스를 제공함
	- Web Service, FTP Service, Mail Service 등등..
-> 프로토콜을 이용하여 통신규약을 정하고 통신을 함
	- Web = HTTP, Mail =SMTP, POP3, IMAP 등...

* 인터넷을 통하여 가장 많이 제공되는 것 중 하나가 바로 Web!!

@ 웹(Web)이란>
-> 네트우크에 연결된 시스템의 "정보" 를 한위치에서 "검색"할수 있는 서비스
-> 1989 년 CERN(유럽 입자 물리 연구소)의 팀 버너스 리에의해 개발됨
-> Hyper Text를 통해 정보를 교환하는 서비스
	-Hyper Text : 하나ㅑ의 웹문서에서 제공하는
	연결 주소(링크)를 통해 다른 문서로 이동 하는 것

@ Web Veirwion
- Web 1.0 : 정적인 웹과 동적인 웹이 있음
	정적 웹( 초기 서비스 버전) - 고정적인 페이지를 제공
	동적 웹( 현재0 가장 많이 사용되는 버전) - 동적인 검색 및 사용자의 참여, 공유 개방

- Web 2.0 : 지능형 웹서비스 (사용자의 패턴을 분석하여 개인별 맞춤 정보 검색
	서비스를 제공)
	- 사용자에게 필요한 지식만을 자동으로 검색하여 보여줌
	- facebook, youtube, instagram 등...

@ Web의 역사
-> 웹은 www라고 부르며 World Wide Web 의 약자
-> 월드 와이드 우[ㅂ은 유럽의 입자물ㄹ학 연구소에서 얻어지는 엄청난 양의 연구
결과 및 자료의 효율적인 공유를 목적으로 1989년 팀 버너스리의 제안으로 개발되었음
-> 1990년도에 웹이 등장하면서 인터넷 또한 급속도로 발전 하기 시작함
-> 웹을 이용하여 데이터를 주고 받기 위해 사용하는 언어는 HTML을 사용함

@ HTML (Hyper Text Markup Language)
-> Web Data(Web Page) 를 전송하기 위해 사용되는 HyperText를 구성하는 Markup Lanugage
	- Hyper Text : Hyper Link를 통해 구성된 형태로 다른 페이지로 쉽게 연결해주
	는 구조로 된 문서
	= Markpup Language(마크업 언어) : 태그 등을 이용하여 문서나 데이터의 구조를
				정의하는 언어.
 
@ HTML의 역사
1. 인터넷의 시작
-ARPANET은 미국에서 소련에 인공위성 발사 성공에 대한 위협을 느껴 새로운 기술을
연구하였고, 이때 인터넷을 만들게 되었음( 군사목적)
- 핵과 같은 공격에 대비하여 전체- 통신 시스템에서 데이터를 안전하게 보관 및 전송
할수 있는 시스템을 만들기 위함
-> 초기 ARPANET이 현재의 인터넷으로 발전 하게 됨

2. 웹의 등장
-> 유럽공동 원자핵 연구소의 팀 버너스리의 제안으로 웹이 개발 되기 시작함
-> 추후 팀 버너스 리는 W3C재단을 창설하게 됨
-> W3C는 웹 표준을 지정하는 기관( HTML 표준을 제정하는 단체)
-> W3C는 HTML 표준을 제정하지만 표준을 강제 하지는 않았음
->  그로 인하여 웹 브라우저 전쟁이 일어나게 됨

3. 제 1차 웹 브라우저 전쟁!!( 두둥)
-> 기존 웹 서비스는 TUI형태 였음
-> 이때, 모자이크라고 하는 최초의 GUI 모습을 갖춘 브라우저가 탄생 함
-> 모자이크는 점점 발전하면서 모자이크 커뮤니케이션에서 넷스케이프 커뮤니케이션으로
이름을 바꾸게 됨
	- 웹 브라우저 명칭도 모자이크에서 넷스케이프로 바꿈
-> 1994 년 넷스케이프는 정식 버전을 발표하고 배포를 시작하게 됨
-> 이에 맞서 마이크로소프트가 인터넷 익스플로러를 발표함
-> 두 회사는 자신들의 웹 브라우저를 사용자가 쓰게 하기 위하여 W3C의 표준을 무시하고
(애초에 W3C가 표준을 강제화 하지 않았으므로!) 자신들만의 엄청나게 많은
기능을 추가해 서로의 웹 브라우저 점유율을 높이기 위해 노력 함

-> 인터넷 익스플로러가 아무리 노력해도 넷스케이프를 이기지 못하게 되자
마이크로소프트 사는 윈도우운영체제에 강제적으로 인터넷 익스플로러를 설치하여
배포 하게 됨(Windows를 사용하게 되면 자동적으로 점유율이 높아지고 사용자도 쓰게 됨)

-> 추가적으로 마이크로소프트는 애픍과의 계약을 통해 매킨토시의 기본 웹 브라우저도
인터넷 익스플로러로 설정함
-> 결국 넷스케이프사는 인터넷 익스플로러의 점유율을 따라 잡기 힘들어지고 1998년에
아메리카 온라인에 인수 됨.

4. 플러그인
-> 웹 브라우저 전쟁으로 인해 웹은 엄청난 속도로 발전하기 시작함
-> 하지만 정작 웹 표준을 지정하는 W3C는 빠르게 대응하지 못함
-> 이에 불만을 느낀 기업들은 플러그인을 만듦
	- 플러그인은 웹 브라우저와 연동되는 특정 프로그램을 사용자 PC에
	추가로 설치해 웹 브라우저 기능을 확장하는 방법
-> 1996년 부터 현재 어도비 플래ㅅ시. 액티브엑스 등은 웹을 점점 풍부한 기능을 가진
공간으로 만들어 줌
-> 그로 인하여 웹 브라우저를 통하여 동영상 시청, 음악 감상 및 금융 업무, 시스템 관리등 다양한 일을 할 수 있게 됨

5. 웹 2.0 시대
-> 2000 년대부터 액티브 엑스를 기반으로 웹 애플리케이션들이 제작 되었음
-> 또한 플래시를 기반으로 일반 사용자를 위한 애니메이션들이 만들어 지게 됨
	- 이때 졸라맨, 우비소년, 마시마로 등이 나오기 시작함
-> 플래시 와 액티브엑스는 점점 발전 하게 됨
-> 인터넷은 점점 사용자끼리 뭉쳐 새로운 컨텐츠를 개발할 수 있는 공간으로 발전하게 되고 사용자끼리 정보 및 컨텐츠를 올려 창조 할 수 있게 된 웹 2.0 세대를 맞이하게 됨

6. WHATWG 의 등장

-> 전 세계 최고 점유율을 가진 인터넷 익스플로러는 사실상 W3C의 표준 웹 브라우저가 됨
-> 그러다 보니 모든 웹 사이트에 액티브 엑스 플러그인이 들어가게 되면서
웹 사이트는 무거워지기 시작함 (시작하기 전에 액티브엑스를 설치 해야 하기 때문!)
-> 이를 막고자 2004년 W3C회의에서 모질라 재단과 오페라 소프트웨어는 새로운
HTML표준 제안서를 W3C에 제출함
	- 그러나 이 제안은 " 웹의 혁명을 위한 기존의 지향점에 위배 됨" 이라는 이유		로 거절 당함
-> 그리하여 인터넷 익스플로러는 더욱 독점을 하게 됨.
-> 인터넷 익스플로러를 제외한 웹 브라우저 제공 기업들(애플, 모질라, 오페라 등) 은 
독자적으로 새로운 웹 표준 기관을 설립함( WHATWG의 등장)
-> WHATWG는 새로운 웹 표준으로 Web Application 1.0을 표준으로 작성하여 만듦(W3C를 무시)
-> 이와 비슷한 시기에 W3C는 마이크로 소프트사와 XHTML2.0 표준을 작성하였고 배포함
-> 그러다 XHTML2.0은 기존의 표준과 너무 동떨어져 있어서 자연스레 개발자들은
XHTML2.0 표준을 사용하지 않게 됨 (2009년 XHTML 2.0 표준은 폐기 됨)
-> 실제 개발자들은 XHTML 2.0을 사용하지 않고 WHATWG 에서 만든 Web Application 1.0을
많이 사용하게 되었고, W3C는 새로운 웹 표준으로 Web Application 1.0을 표준으로 선택함 ( 마이크로 소프트 버림...)
-> 결국 W3C는 Web Application 1.0 표준을 HTML5 표준으로 변경하고 WHATWG와 함께
HTML5 를 발표함
-> HTML5 표준은 2012년에 초안을 발표하고 2014년 10월에 정식 권고안을 발표하였음.

7. 제 2차 웹 브라우저 전쟁
-> 결국 마이크로소프트와 W3C가 함께한 XHTML 2.0 표준이 붕괴되고 인터넷 익스플로러의
기능 문제가 대두 되었음
	- 다른 웹 브라우저는 모두 최신 표준을 지원하는데 인터넷 익스플로러는 오히려
	최신을 지원하지 못하는 역현상이 발생함
-> 한마디로 지금까지의 웹 브라우저 점유율을 뒤집을 수 있는 기회가 생겨남
	( 웹 브라우저 업체들은 지금이 기회다 싶음!)
->  이 시기를 놓치지 않으려고 모든 웹 브라우저가 빠른 속도로 웹 브라우저 
업데이트를 진행 함
	- 모질라의 파이어폭스는 3년에 한번 하던 브라우저 버전업을 2달 단위로 하고있음
	- 구글의 크롬은 10주 단위로 버전업이 되고 있음
-> 실제 웹 브라우저 기업들은 개발 외에도 여러 마케팅 까지 하게 됨
	- 국내에는 인터넷 익스플로러가 점유율이 높아 다른 웹 브라우저 기업이 마케팅
	하지 않으나 옆ㅎ나라 일본 같은 경우에는 유명한 캐릭터들을 앞세운
	다양한 광고를 하고 있음
-> 심한 경우에는 해당 브라우저가 아니면 서비스에 접근하지 못하는 경우도 만들기도 함
-> 현재 제 2차 웹 브라우저 전쟁이 지속되고 앞으로도 언제까지 할지는 모름
	(어떠한 변화가 발생할지 모르고 웹 개발자들은 그 변화를 겪어야 될 수 밖에 없음)


@ HTML 구성요소

-> 설명을 위해서 P 태그를 사용했을 뿐 P 태그를 정의하는게 아님을 기억하세요.

<p align = "center"> HTML/CSS</p>

<p : 태그의 시작
allgn : 속성
"center" : 속성값
HTML/CSS : 내부문자 (출력되는 부분)
</p> : 태그의 종료

태그 (Tag) : '<' 와 '>' 로 붂인 명령어
	시작 태그(<태그>) 와 종료태그( </태그>) 를 한쌍으로 이용

요소 : 시작태그와 종료태그로 이루어진 모든 명령어
	하나의 HTML 문서는 요소들의 집합
속성 : 요소의 시작 태그에만 사용( 종료태그에는 사용 X)
	여러개의 속성을 사용할 수 있으며 속성의 구분은 공백으로 구분함

변수 / 속성값 ; 속성이 가지는 값, 값 입력시 "" 혹은 '' 를 이용함

@ html 기본구조

<!doctype html> : 문서유형
<html> : html 의 시작
<head> 
	문서의 각종 정보와 문서 자체에 대한 설명 내용
	<title> ,<meta>, <link> , <style> ,<script> 등을 사용함
</head>
<body>
	화면에 출력해서 보여주는 정보 및 내용
	head에 들어가는 태그를 제외하고 모든 태그를 사용
</body>
</html>	 : html의 끝
